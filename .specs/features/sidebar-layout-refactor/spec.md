# Spec: Sidebar Layout Refactor

## Goal
Implement a premium, unified Sidebar navigation system for both Administative and Distributor areas, replacing the current scattered navigation with a state-of-the-art layout that adapts to functional roles.

## Core Requirements

### 1. Structure (shadcn/ui Sidebar)
- **Framework**: Use the official `shadcn/ui` Sidebar component.
- **Composition**:
  - `SidebarHeader`: 
    - **Logo**: Product Logo (Task 2).
    - **Fallback**: Display initials or skeleton placeholder (Skeleton) during loading.
  - `SidebarContent`: NavItems grouped by logic.
  - `SidebarFooter`: User profile summary & Logout button.
  - `SidebarRail`: For resizability/collapse.
- **Behavior**:
  - **Collapsible**: `collapsible="icon"` (Radix).
  - **Persistence**: Save `isCollapsed` state in `localStorage` for cross-session consistency.


### 2. Navigation Items (Logical Mapping)

#### Admin Roles (Internal area)
| Item | Icon | Destination | Context |
| :--- | :--- | :--- | :--- |
| **Home** | `Home` | `/admin/dashboard` | Task 6 (Metrics) |
| **Parceiros** | `Users` | `/admin/dashboard/partners` | Partner Management & Approvals |
| **Catálogo** | `Package` | `/admin/dashboard/products` | Task 7 (Product Editing) |
| **Stocks** | `FileUp` | `/admin/dashboard/stock` | Task 10 (Routing) |
| **Encomendas** | `ShoppingCart` | `/admin/dashboard/orders` | Sales overview |
| **Definições** | `Settings` | `/admin/dashboard/settings` | Task 5 (Admin Profile) |

#### Partner/Distributor Roles (Dashboard area)
| Item | Icon | Destination | Context |
| :--- | :--- | :--- | :--- |
| **Home** | `Home` | `/catalog` | Portal Product Catalog |
| **Comprar** | `ShoppingBag` | `/catalog` | Product Listing |
| **Checkout** | `ShoppingCart` | `/checkout` | Task 4 (Order Attributes) |
| **Histórico** | `ClipboardList` | `/orders` | Task 3 (Purchase History) |
| **O Meu Perfil** | `Building2` | `/profile` | Tasks 1 & 2 (Profile/Logo) |

### 3. Responsive Behavior
- **Desktop**: Sidebar fixed to the side, potentially collapsible.
- **Mobile**: Hidden by default, toggled via a Hamburger menu into a **Sheet (Overlay)** component.
- **Rules**: Must be generated by **Stitch MCP** for both viewports.

- **Desktop Visibility**: Verify sidebar is visible on load.
- **Responsive States**:
  - **Expanded**: Verify labels and icons are visible.
  - **Collapsed**: Verify only icons are visible; labels must be hidden. Verify `localStorage` persistence after refresh.
